# PRD Alignment Roadmap

## Goal
Transform the current PMS into a fully PRD-aligned AI-native performance platform with seamless lifecycle UX, explainable decision intelligence, and enterprise governance.

## Delivery Strategy
The PRD scope is product-scale and must be delivered in controlled phases to protect stability and data integrity.

## Phase 0: Alignment and Foundations (In Progress)
- Normalize module map against PRD requirements.
- Define source-of-truth lifecycle objects (cycle, framework, timeline nodes, approvals).
- Add implementation checklist and acceptance criteria.

Exit Criteria:
- Gap matrix complete.
- Prioritized backlog finalized.
- Ownership per module clear.

## Phase 1: Performance Lifecycle Core
- Performance framework engine (OKR, MBO, Hybrid and extensible options).
- Cycle configuration (quarterly, annual, hybrid) with HR/Admin control.
- Single timeline lifecycle model and API contract.
- Auto-lock states and next-step guidance in UI.

Exit Criteria:
- Employee flow works end-to-end for goal create -> approval -> check-in enablement.
- Timeline state machine validated.

## Phase 2: Advanced Goal System
- AI goal suggestions using role, title, department, AOP context, and external role intelligence adapters.
- Goal cascading and drag-and-drop allocation with contribution percentages.
- Goal lineage graph (employee -> manager -> business objective).
- Drift detection and goal change log.

Exit Criteria:
- Goal lineage visible to employee and manager.
- Cascading and weight normalization operational.

## Phase 3: AI Check-ins and Ratings Engine
- Calendar-first check-in scheduling with availability lookup and mandatory integration flow.
- Transcript ingestion and per-goal summary auto-population.
- Manager feedback coaching with tone scoring and structured suggestions.
- Rating recommendation with explainability and manager override.

Exit Criteria:
- Check-ins, transcript summaries, and rating suggestions production-ready.
- Employee visibility rules strictly enforced.

## Phase 4: Reviews, Dashboards, and Decision Intelligence
- Quarter/year AI narrative reviews.
- Manager dashboard with stack ranking, risks, and heatmaps.
- HR control center: calibration, audit, 9-box, readiness and TNA.
- Leadership command center with aggregated decision-safe insights.

Exit Criteria:
- Persona-specific dashboards complete.
- Exports and strategic summaries operational.

## Phase 5: Governance, Security, and Cost Control
- AI usage caps by feature and persona with transparent counters.
- Full audit trail and explainability metadata.
- Compliance hardening (GDPR, DPDP controls and retention policy).
- Notification orchestration and timeline reminders.

Exit Criteria:
- Security and governance checklist completed.
- Usage cap policy enforceable and observable.

## Phase 6: Final UX Polish and Rollout
- Navigation buddy and in-app guidance.
- Friction removal and copy harmonization by persona tone.
- Beta program, telemetry review, and AI quality tuning.

Exit Criteria:
- Adoption, completion, and manager participation targets trending to PRD metrics.
- Production rollout approved.

- configurable cycle windows
- framework selection rules
- timeline status transitions
